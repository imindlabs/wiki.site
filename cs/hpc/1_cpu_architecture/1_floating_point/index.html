1.1 Floating Point

IEEE754

The IEEE 754 standard defines the representation and behavior of floating-point numbers in computing. This standard is widely adopted across different computer architectures and programming languages to ensure consistency in floating-point arithmetic. Here’s a summary of how IEEE 754 applies to float (single-precision) and double (double-precision) floating-point numbers:

IEEE 754 Standard Overview

  1. Floating-Point Formats:

    • Single Precision (32-bit):

      • Sign Bit: 1 bit
      • Exponent: 8 bits
      • Fraction (Mantissa): 23 bits

    • Double Precision (64-bit):

      • Sign Bit: 1 bit
      • Exponent: 11 bits
      • Fraction (Mantissa): 52 bits

  2. Representation:

    • Sign Bit: Determines whether the number is positive or negative.
    • Exponent: Encodes the magnitude of the number, using a bias to allow for both positive and negative exponents.
    • Fraction (Mantissa): Represents the significant digits of the number, with an implicit leading bit for normalized numbers.

  3. Special Values:

    • Zero: Represented by all exponent and fraction bits being zero, with the sign bit determining the sign of zero.
    • Infinity: Represented by all exponent bits set to 1 and all fraction bits set to 0. Positive and negative infinities are distinguished by the sign bit.
    • NaN (Not-a-Number): Represented by all exponent bits set to 1 and a non-zero fraction. NaNs are used to represent undefined or unrepresentable values (e.g., 0/0).

  4. Rounding Modes:

    • Round to Nearest (default): Rounds to the nearest representable value. If the value is exactly halfway between two representable values, it rounds to the nearest even number.
    • Round Toward Zero: Rounds towards zero (truncation).
    • Round Toward Positive Infinity: Rounds towards positive infinity (ceiling).
    • Round Toward Negative Infinity: Rounds towards negative infinity (floor).

  5. Precision and Accuracy:

    • Single Precision (32-bit): Provides approximately 7 decimal digits of precision.
    • Double Precision (64-bit): Provides approximately 15-16 decimal digits of precision.

Example of IEEE 754 Single Precision Representation

To illustrate, let’s represent the number -13.25 in IEEE 754 single-precision format:

  1. Convert to Binary:

    • Decimal -13.25 in binary is -1101.01.

  2. Normalize Binary:

    • Normalize to -1.10101 × 2^3.

  3. Encode Components:

    • Sign Bit: 1 (since the number is negative)
    • Exponent: 3 + 127 (bias) = 130 → Binary 10000010
    • Mantissa (Fraction): The binary fraction 10101 is represented in the 23-bit mantissa field, with trailing zeros added to fill the field.

    Combining these:

    • Sign Bit: 1
    • Exponent: 10000010
    • Mantissa: 10101000000000000000000

Summary

The IEEE 754 standard defines how floating-point numbers are represented, including format, precision, special values, and rounding modes. For float (single-precision) and double (double-precision) floating-point numbers, the standard ensures consistent and predictable behavior across different hardware and software implementations.

Float Operations in CPU

Float variable vs Float Register

Storing a float value in a variable versus storing it in a floating-point register involves subtle differences in how precision is managed and maintained. Here’s a breakdown of the differences:

1. Floating-Point Registers:

  • Precision Handling: Floating-point registers in modern CPUs (such as x86 and ARM architectures) are designed to handle floating-point operations with high precision. These registers usually adhere to the IEEE 754 standard for floating-point arithmetic.

  • Intermediate Calculations: During calculations, floating-point registers maintain the precision of the intermediate results as much as possible according to the precision of the floating-point format (e.g., 32-bit float or 64-bit double). These registers can hold intermediate results with precision but may still be limited by the precision of the float type.

  • Temporary Storage: Floating-point registers are used temporarily during arithmetic operations. The precision of intermediate results in these registers is generally consistent with the precision limits of the float type. The precision of the result can be affected by rounding errors in operations.

2. Storing in a Variable:

  • Precision Limits: When a floating-point result is stored in a float variable, it is subject to the precision limits of the float type. For example, in a 32-bit float, the result is stored with about 7 decimal digits of precision.

  • Variable Storage: Variables in memory (RAM) also adhere to the precision of the float type. When a floating-point value is stored in a variable, it is stored in memory with the precision determined by the float format.

  • Possible Additional Rounding: The process of storing a floating-point result in a variable may involve additional rounding or truncation, especially if the result is computed in a higher precision format (such as double) and then stored as float.

Key Differences in Terms of Precision:

  1. Intermediate Precision:

    • Registers: Floating-point registers can handle intermediate results with precision consistent with the floating-point format (e.g., 32-bit or 64-bit). The precision is maintained during calculations but is limited by the format used.
    • Variables: Precision is maintained according to the format of the variable (float or double). When the result is stored in a float variable, it is stored with the precision of a float, which might involve rounding if the computation was performed with higher precision.

  2. Operations Impact:

    • Registers: Operations performed directly in floating-point registers maintain precision as much as possible during computation. Precision loss occurs during the computation itself due to rounding.
    • Variables: Storing results in a float variable means the result is subject to the precision limits of float. If intermediate calculations were performed in higher precision, the result might be rounded to fit the float format when stored in a variable.

Summary:

  • Floating-Point Registers maintain intermediate precision consistent with the floating-point format used for calculations. They handle precision during computations but are limited by the floating-point format (e.g., 32-bit float).
  • Storing in Variables means the result is held with the precision of the floating-point format (float), and additional rounding might occur if the result was computed in a higher precision.

Both methods are constrained by the precision limits of the float type, but floating-point registers handle precision during computations, while variables store the final result with precision limits of the float format.

FPU

The precision maintained by the Floating-Point Unit (FPU) during arithmetic operations can vary depending on the architecture and the floating-point format used. Here’s a breakdown of how precision works in the context of FPUs and floating-point formats:

Precision in Floating-Point Units (FPUs)

  1. 32-bit float Precision:

    • When working with 32-bit float values (single precision), the FPU performs arithmetic operations with the precision of the 32-bit format. This includes 23 bits for the fraction (mantissa) plus an implicit leading bit, and 8 bits for the exponent.

  2. Intermediate Precision:

    • Extended Precision: Many modern CPUs use extended precision during intermediate calculations to reduce rounding errors. For example, x86 processors often use an 80-bit extended precision format internally. This format includes:

      • Sign Bit: 1 bit
      • Exponent: 15 bits (with a different bias than the 32-bit format)
      • Fraction (Mantissa): 64 bits

    • 80-bit Extended Precision (x87 FPU): In x86 architecture, the x87 FPU uses an 80-bit extended precision format for intermediate calculations. This allows more precise results during arithmetic operations, reducing the impact of rounding errors before rounding to the final 32-bit or 64-bit format.

  3. Final Rounding:

    • To Target Format: After intermediate calculations, results are rounded to fit the precision of the target floating-point format (float, double, etc.). For instance, when storing a result computed in extended precision (80-bit) into a 32-bit float, the result is rounded to fit the 32-bit format.

Example of Precision Handling

  • Final Rounding: The result of the multiplication is rounded to fit into the 32-bit float format before being returned.

Summary

  • 32-bit float Operations: When using 32-bit float values, the precision of individual operations is 32-bit. However, intermediate calculations might use extended precision (e.g., 80-bit on x86) to improve accuracy.

  • Intermediate Precision: Modern CPUs often use more precise formats for intermediate calculations to reduce rounding errors before rounding to the final 32-bit or 64-bit format.

In conclusion, while 32-bit float operations are performed with 32-bit precision, the FPU may use extended precision for intermediate results to improve accuracy during computations.

Example

When I say “maintain precision as much as possible during computation,” I mean that floating-point registers are designed to perform arithmetic operations with a high degree of accuracy consistent with the floating-point format they support. However, this precision is still constrained by the format (e.g., 32-bit float), and rounding errors can occur.

Here’s a detailed comparison between floating-point registers and floating-point variables in terms of precision handling:

Floating-Point Registers

  1. Intermediate Precision:

    • Arithmetic Operations: Floating-point registers handle arithmetic operations (addition, subtraction, multiplication, division) with precision consistent with the floating-point format they support (e.g., 32-bit float or 64-bit double).
    • Operations: While performing operations, the floating-point unit (FPU) tries to maintain as much precision as possible given the format’s constraints. For example, the result of a multiplication or division is computed with the precision allowed by the floating-point format (e.g., 24-bit significand for float).
    • Intermediate Results: Intermediate results are kept with the precision of the format used, but they might not be exact due to rounding errors inherent in floating-point arithmetic.

  2. Final Rounding:

    • Conversion to Memory/Variable: When an intermediate result is finalized and needs to be stored in memory or a variable, it is rounded to fit the precision of the destination format. For instance, if the final result of a computation in a float register needs to be stored in a float variable, it will be rounded to fit the 32-bit float representation.

Floating-Point Variables

  1. Precision of Storage:

    • Precision Limits: A floating-point variable stores a value with the precision defined by its type (e.g., 32-bit float or 64-bit double). The precision is fixed at the time of storage.
    • No Extra Precision: Once a result is stored in a float variable, it adheres strictly to the precision of the float format (e.g., approximately 7 decimal digits). There is no additional precision beyond this format.

  2. Additional Rounding:

    • Result Handling: If a result was computed with higher precision (e.g., in double registers), storing it in a float variable introduces additional rounding to fit it into the 32-bit precision. This can result in rounding errors if the result was initially computed with higher precision.

Summary

  • Registers: Floating-point registers maintain precision during intermediate calculations according to the format’s constraints and the operations performed. Rounding occurs naturally during calculations but is managed within the precision limits of the floating-point format.

  • Variables: Floating-point variables store values with a fixed precision. Additional rounding may occur when storing results computed with higher precision into variables of a lower precision type.

Both methods are constrained by the precision of the floating-point format, but floating-point registers handle intermediate results with precision consistent with the format used during computation, while floating-point variables store the final result with their fixed precision.
